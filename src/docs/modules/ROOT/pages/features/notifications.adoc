:description: Guide to workout reminder and streak notifications in RepFoundry.

RepFoundry can send scheduled push notifications to remind users to work out on chosen days and to protect active training streaks.

== Overview

Notifications are delivered via the `flutter_local_notifications` package using timezone-aware scheduling from the `timezone` package.
`NotificationService` is a singleton that manages all scheduling.
User preferences are stored in `ReminderSettings` and persisted via SharedPreferences through `ReminderSettingsNotifier`.

=== Scheduling Weekly Reminders

[source,dart]
----
Future<void> scheduleWeeklyReminders(ReminderSettings reminderSettings)
----

Cancels all existing reminders, then schedules one notification per enabled day using `FlutterLocalNotificationsPlugin.zonedSchedule()`.

Notifications repeat weekly (`matchDateTimeComponents: DateTimeComponents.dayOfWeekAndTime`).

=== Scheduling the Streak Reminder

== Scheduling Algorithm

The `_nextInstanceOfDayAndTime` helper finds the next calendar occurrence of the target `(weekday, hour, minute)` tuple in the device's local timezone.
If the computed time has already passed today, the schedule is advanced by 7 days.
The `_nextInstanceOfTime` helper performs the same calculation for the daily streak reminder without a weekday constraint.

== Android Permissions

Declare the following in `AndroidManifest.xml`:

[cols="1,2"]
|===
| Permission | Purpose

| `POST_NOTIFICATIONS`
| Required on Android 13+ (API 33) to show notifications

| `SCHEDULE_EXACT_ALARM` or `USE_EXACT_ALARM`
| Required for exact alarm scheduling; `inexactAllowWhileIdle` is used for battery efficiency but still requires the permission on some OEM configurations

| `RECEIVE_BOOT_COMPLETED`
| Re-schedules notifications after device restart
|===

NOTE: RepFoundry uses `AndroidScheduleMode.inexactAllowWhileIdle` for all reminders.
This balances battery life against delivery reliability; reminders may fire a few minutes late during Doze mode.

== iOS Permissions

iOS permissions (`alert`, `badge`, `sound`) are requested at initialisation time via `DarwinInitializationSettings`.
No additional `Info.plist` keys are required beyond those added by the `flutter_local_notifications` package.

== Settings UI

The notifications section in the Settings screen contains:

* *Day picker* â€” a row of toggle chips for Monday through Sunday
* *Time picker* â€” opens the platform time picker; updates `hour` and `minute`
* *Streak reminder toggle* â€” enables or disables the daily streak protection notification

Changes are saved immediately via `ReminderSettingsNotifier`, which reschedules all active notifications to reflect the updated preferences.
